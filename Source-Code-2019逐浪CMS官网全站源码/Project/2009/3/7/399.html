解决方案背景

 

门户网站群是一个由系列网站构构成的应用系统，其应用特征包括：
拥有多种复杂的应用逻辑，如文章发布、下载信息、视频点播
拥有多个独立的模块或关节键点，如中心局和下属局间的关心
需要有严格的权限管理，如下一级不能管理上一级权限
需要自由分配权限的功能，如增加下一级权限
可自由派生子站，如中心单位增加一个子单位
可以为子站分派独立域名
各个子站间拥有独立的管理权限以及增进的办公应用需求
敏捷的信息发布需求
方便的信息检索与管理应用

面对如此复杂的应用体系，普通的CMS是无法完成相关的应用，Zoomla!逐浪CMS根据多年的服务于大型企业客户与政务系统的经验，通过长达近四百个工作日的研发，于1.0版本中正式支持门户网站群与子站系统的高端应用。

注：本方案简称“门户网站群解决方案”，全称“Zoomla!逐浪CMS门户网站群与子站系统解决方案”。

逐浪CMS解决方案的优势

 

在逐浪CMS中，门户网站群的功能主要通过节点、权限、角色、多元转换四个体系来实现，构建起了一个四维的立体应用体系。
 

节点
可控制子站转发项，一键即可从从属节点转为独立门户
角色
针对不同的节点设定不同的权限，通过层层的“密室”功能，来建立起独立的商业门户。
权限
支持面向对象开发的权限功能，不同的专题、栏目、节点应用不同的环境部署，以满足子站用户的需求。
模型
多模型管理，同一节点可以绑定多个功能模型，从而使子站系统的二次开发成为可能。
多进程 处理机制
多子站可以在线处理相关事务，而互不干扰。
Web Services开发支持
支持面向高级的扩展开发

Zoomla!逐浪CMS的Web Service应用系统，是集成开放式计算理念，为了使原来各孤立的站点之间的信息能够相互通信、共享而提出的一种接口。
如同标准 Web Service所使用的是Internet上统一、开放的标准，如HTTP、XML、SOAP（简单对象访问协议）、WSDL等，所以Web Service可以在任何支持这些标准的环境（Windows,Linux）中使用。注：SOAP协议（Simple Object Access Protocal,简单对象访问协议）,它是一个用于分散和分布式环境下网络信息交换的基于XML的通讯协议。在此协议下，软件组件或应用程序能够通过标准的HTTP协议进行通讯。它的设计目标就是简单性和扩展性，这有助于大量异构程序和平台之间的互操作性，从而使存在的应用程序能够被广泛的用户访问。
面向服务的体系结构（Service-Oriented Architecture，SOA）是一种 IT 体系结构风格，支持将您的业务转换为一组相互链接的服务或可重复业务任务，可在需要时通过网络访问这些服务和任务。这个网络可以是本地网络、Internet，也可以分散于各地且采用不同的技术，通过对来自纽约、伦敦和中国香港的服务进行组合，可让最终用户感觉似乎这些服务就安装在本地桌面上一样。可以对这些服务进行结合，以完成特定的业务任务，从而让您的业务快速适应不断变化的客观条件和需求。

而这些对于门户网站群系统的应用尤其显得重要，Zoomla!逐浪CMS将这些复合的应用技术融合为一体，构建功能强大的门户网站群功能。

进入逐浪CMS的后台-系统配置-节点管理，在这里，我们可以自由的定义节点的权限和逻辑。
节点中提供子站选项，如果启用子站，则可以将子站进行域名转发设置。
如：主站使用的是www.aaa.com，子站则可使用other.aaa.com或www.other.com等域名。

可应用范围

应用场景一：大型企业事业单位，通过子站系统，可以为各个分支机构提供独立网站的建设。同时由于采用了统一数据库和系统配置，成本大大降低，管理应用方便，无需重复投资，即可完成部署。

应用场景二：教育系统、政务系统、水电矿局等公办事业单位，可能通过这样的平台，来快速聚合、共享信息。综合的信息，使资源得到了良好的利用，便于管理。

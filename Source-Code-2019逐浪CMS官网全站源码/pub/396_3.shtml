详细说明

在网站的建设中，栏目与频道的构建，就如同的人类肢体一样的重要，因为它是用来装载网站的重要路管道。
在传统的CMS系统中，一般采用栏目、频道、节点这三个种方法来处理，此三者构成网站的纵向路径。而专题则构成了网站的横向路径。
于是一张双向的网织成了，经纬俱全，看上去很美。

可是，富有经验的站长都知道，仅仅只有两种结构，对于复杂的网站来说，是远远不够的，基至有时还限制了网站的成长。

Zoomla!逐浪CMS认识到网站未来的需求和高端用户的扩展性应用，从立项伊始就定义了其节点开发的原则：
无限扩展
无限张别
可自由迁移
可多节点导步传输容纳数据。

自由洒脱的功能设计

首先，在逐浪CMS中，节点的功能是不限受限制的。管理员可以建议任意个节点，并在任意层次添加节点，无限层次节点的设计，保证了网站结构不受约束。

 

超载传统的组织形式：其次，传统网站的节点事实上只有“频道”、“栏目”两种结构，Zoomla！逐浪CMS大大扩展了这一应用，每一个节点都提供了列表页和节点首页的功能，这样的应用，可以让网站自由的进行频道与栏目的转换，使管理员在处理网站结构时得心应手。

有经验的站长，都会有一个头痛的地方，那就是网站的应用往往受限于功能，比如用户需要在一个名为“本地新闻”中发布一条全图片的信息，如果采用普通文章功能发表显然很吃力、显不直观。

多功能的节点设置：在Zoomla!逐浪CMS中，节点被赋予了功能组织的角色，用户可以在节点管理中进行不同的功能绑定设置，与Zoomla!逐浪CMS的功能模型相结合，可以绑定不同的功能，节点仅仅是信息的一种组织方式，而不再是功能的桎梏。

子站功能让节点化蛹成蝶：Zoomla!逐浪CMS的频道变子站和新子站派生功能，是针对大型网站设计的专业功能模块。
通过此功能，用户可以自由的将节点快速的转为拥有独立域名的子站，同时由于采用了最新的技术解决方案，用户不需要进行二次部署即可成功实施子站拓展，省下了大笔的服务器、空间与带宽费用，为经营腾出了宝贵的空间。

丰富的自设内容让节点容颜焕发：Zoomla!逐浪CMS的开发者们认识到，一个卓越的网站，其节点需求也是多样的。为了满足网站的经营需求，逐浪CMS提供了自由的节点选项设计，用户可以通过“自设内容”功能来扩展节点信息，使节点本身也具备装载丰富多样的、支持超文本标记（HTML）的信息。

通过Zoomla!逐浪CMS的节点应用，用户还可以对信息的生成管理进行设置，无论是生成传统的HTML静态，还是生成服务器端的STHML文件格式，都会得心应手。同时对评论管理进行了严格的筛选，结合节点设计，以满足权限的管理。

国内首创独有的中文名生成机制，为站长SEO和信息的特殊处理提供了最优化的质量选择--用心关注站长的每一个细微需求，已经融入了Zoomla!逐浪CMS的每一个开发细节中，必求精品的开发理念，保证了网站的应用

如同渔夫织网一样，节点是经、专题是纬，Zoomla!逐浪CMS开发者力求为用户打造牢固的系统平台。

详细说明

关键字：游标、批处理、视图

 

游标是邪恶的！

在关系数据库中，我们对于查询的思考是面向集合的。而游标打破了这一规则，游标使得我们思考方式变为逐行进行.对于类C的开发人员来着，这样的思考方式会更加舒服。

如同在ATM取款的时候，是一次取1000元效率更高呢，还是分10次取100元效率高？

让人悲催的是，浮标的计算方法却是采用分10次取100次元的方法，那是因为在实际的使用中，有许多的特殊需求需要通过这些更笨的方法来完成。

比如在Zoomla!逐浪CMS实施的一个国内著名的大型分成返利系统中，需要每天定定时对代理商进行分成，同时分成的基数又要根据每天下线的销售总额进行统计从而计算出新的分成系数，同时分成的人员名单又是不断的变化，同时销售总额还有根据不同的会员进行个性化的加权处理，而且所有行为都必须第一时间自动完成，一般的系统则面对这样的复杂的处理，只好爱莫能助了。

而在Zoomla!逐浪CMS2中，全新的开发中心，提供了一系列的增进处理。

 

&ldquo;开发中心&rdquo;这个模块是从Zoomla!逐浪CMS2中向广大朋友开放的，其设计出发点是试图窥视对大数据处理的更加便捷的处理体制，所以从最初的简单的数据查询和处理，扩展到了全新的开发中心，力图能够对整个系统的数据进行处理，满足系统的高端需求。

我们无法否认，仅仅凭传统的CMS造单薄的标签和模板，很多时候并不能完成我们所需要的系统开发，而网站系统越来越多的智能化，甚至连简单的数据表都不能满足我们的查询，人们需要有更加方便的应用处理机制，于是定义于新一代CMS的&ldquo;开发中心&rdquo;应运而生了。

 

（上图：功能完善的开发中心）

 

在整个开发中心，我们提供了数据库全库概况、备份还原数据库、索引优化、服务器信息总览等一系列开发工具，从而满足开发者的高级需求。

 

普通人以为开发中心认为只能进行一些数据库的查询，与实际使用相远，其实她还提供了包括视图管理一样的DBA应用，如我们对一些表名记不住，可以自己命名，并只抽出自己想要的数据，形成个性化的视图：ZL_view_小末的表，这个表一旦创建，查询的时候就更加方便，只要查询自己想要的数据即可：

（上图：创建视图的界面）

整个系统遵循大数据处理的优化，一步式的DBA，满足对网站核心数字资产的重点管理。

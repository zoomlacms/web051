逐浪技术堂：FarmVet Systems 使用 SQL Server 与 Visual Studio 开发移动数据应用，提升农场兽医效率

作者：Q葩小玉 发布时间：2017-03-08 来源:佚名 点击数： 分享网址

博主：Q葩小玉
个人座右铭：你想过普通的生活 就会遇到普通的挫折 你想过上最好的生活 就一定会遇上最强的伤害 这世界很公平 你想要最好 就一定会给你最痛~

作者按：2017年微软公司发布了Visual Studio 2017开发套件，被中国开发者誉为“宇宙第一开发神器”，曾几何时，微软开发工具不被重视，当人们换了几部小米手机、几台安卓平板后，回过一看，从win7到win10，已经过去了10年（windows 7在2008年发布）。

这10年间，以微软为代表的传统软件公司不是毫无作为，而是在生产力平台突飞猛进，并取得了长足的进步。

在BI商业智能、智能硬件、虚拟增强现实、商业智能和大数据上，微软平台的进步引人注目，并成为移动平台的效仿对象。诸如扁平化、云计算、大数据，无不活跃着这样的平台。


Zoomla!逐浪CMS作为生产力平台软件的代表，在WEB内核领域始终引领市场，当国内同行产品还跑在.net2.0框架之上，逐浪软件官方就果断向win2012、win2016高端内核进军，专注于生产力平台，坚持用win平台和MSSQL数据库来致胜。


尤其是最新发布的逐浪CMS2 x3.9.0，在OA、办公、商务、站群、微信、电商平台，均活跃着他们的影子，是国内高端门户与平台建设的首选！


本期引用一篇大数据文章，来诠释这种技术的致胜性。


“农场兽医们是移动工作者，他们渐渐地提高对智能手机和平板的仰赖，以及为了让 VetIMPRESS 变成他们生活中的一部分（就像他们每天生活中会使用的听诊器和威灵顿靴子），这个解决方法必须能在手中的移动设备执行。 ”

－ George Brownlee : 创办人、CEO




随着农场的规模越来越大以及养的牲畜越来越多，农夫们必须花费更多心力在提升农场整体的生产力。 他们需要知道如何通过微小的改变带给农场更大的进步。 例如，相较于请兽医执行一些像是怀孕扫描或是蹄修剪等基本任务，现今的农夫们转而向兽医咨询动物的健康和福利以帮助他们提升农场的效率和生产力。 因此，兽医们不得不改变商业模式。 与其仅仅依赖他们的双手，不如更加运用他们的专业知识。


为了因应这项改变，兽医们需要数据。 他们需要在造访农场之前，了解农场的表现以及动物们的健康状况与生产力。 他们也需要知道：随着时间的推移，农场的表现如何变化，以及它与更广泛之行业的关连。 然而众所皆知，农场的数据是非常零碎的，散落各处及未连接的销售点系统、牛奶加工目录、诊断实验室和整个行业的政府监管数据库。 就连个别的兽医实验中的数据也往往非常零散。

而以上就是这位年轻兽医 George Brownlee，在爱尔兰的 Belfast 服务牲畜农场和奶酪农场并开始他的实验时所经历的状况。

“为了提供我想要的服务水平给我现代农场的客户，我在要去农场的前一天晚上熬夜到凌晨两点钟，必须去寻找各种网站以求得正确合适的信息。 或甚至在前一天到农场去收集计算机上的文件或是从书中获得一些数据。 我必须在 Excel 中做大量的数据分析，把它们印出来，然后隔天再带着这些文件跟着我到农场去。 ”

Brownlee 想要创造一个方式收集行业数据，整合农场数据，并且可以带着它到处巡视。 他想要替他所照料的动物们创造电子医疗病例，以及帮每个农场创造履历简介。

所以在 2010 年时，他与北爱尔兰的另一名兽医合作成立 FarmVet Systems，一个独立的软件贩卖商。 并且创造了 VetIMPRESS，一种安全、以网络为基础的软件即服务解决方案，帮助兽医们收集、标准化，以及分析和它们实验中动物的健康以及福利有关的数据。


进入更进阶的领域

为了提供照料大型动物的兽医和其他在不同农场间游移的服务供应者真正有效的解决方案，FarmVet Systems 的创立者知道他们需要一个可以在任何地方、任何设备上存取进入的方法。 “农场兽医们是移动工作者”FarmVet Systems 的创立者兼 CEO George Brownlee 说到。 “他们渐渐地提高对智能手机和平板的仰赖，以及为了让 VetIMPRESS 变成他们生活中的一部分（就像他们每天生活中会使用的听诊器和威灵顿靴子），这个解决方法必须能在手中的移动设备执行。 ”

通过创造一种移动数据的解决方案，VetIMPRESS 让兽医在现有数据上收集到最多数据变得更简单，并创造一种独特的数据集让他们可以在与客户咨询业务时使用。 FarmVet 希望兽医们和其他农场服务提供商能够轻松的撷取数据，以便立即进行现场分析并告知他们诊断测试的决定。 帮助他们辨识有可能影响农场生产力和效率的潜在问题，以及增加农场顾客端的可见性。

为了替 VetIMPRESS 软件建造一个移动 App，一个在 FarmVet Systems 的小开发团队（包括四位在数据端和三位在应用程序端）使用微软 Visual Studio 和 Visual Studio Tools for Apache Cordova。 这样一来，团队就可以使用单一链接库以定位给 Android、iOS 和 Windows 等多种平台。


“我们需要某种可以尽可能的交互越多平台的方法”Brownlee 说。 “但是我们只有一个小团队。 我们需要可以适应内部技能的开发工作，因为这将会是我们产品提供的重要核心的一部分。 只是我们已经没有办法雇用更多拥有其他技能的人才。 使用 Visual Studio 意味着我们在建造这三个平台时可以省下百分之四十或更多的成本。 ”

 

给小团队的大工作

使用 Visual Studio，开发团队不只可以减少成本，也可以减少上市所需时间。 还可以避免特定平台的编码，代表开发团队可以避免一些如果尝试替不同类型的设备将相同的功能带到市场时会造成的问题（包括除错和顾客支持等问题）。

因为开发工作被简化了，团队持续在改进顾客的用户体验。 兽医可以进入他们的日志，在给定日期的时间表上更新农场动物的纪录，以及存取农场的历史造访纪录。 不论何时，只要设备位于 Wi-Fi 提供的范围内，App 会自动捕捉从微软SQL Server 2012 R2 服务器传来的新信息并且推送更新给兽医。 除了可以撷取动物数据，这个 App 还可以传送兽医的实验室工作并且将报告传给顾客。 并且，当数据流入，他会监视目标和门坎，在准备好被查看的特定数据传入时会提醒兽医。

“我们能够通过 Visual Studio 所建置出来的成品是非常广泛的”Brownlee 说。 “兽医们需要足够的数据以支撑一整天的工作，像是有几千只动物需要照料等等，以及当他们外出、写报告和储存影像时也要产生更多的数据。 Visual Studio 在 App 如何到处传输数据这方面，让整件事变得真的非常有效率。 ”

VetIMPRESS 的顾客数量，一直不断的在成长。 现在还包括药剂师、食品公司以及会使用到农场数据的其他组织。 现在，他们已经可以存取过往不可用或未纪录的农场和动物数据，以创造全面的动物健康和生产力愿景，提供他们的顾客更好的服务。

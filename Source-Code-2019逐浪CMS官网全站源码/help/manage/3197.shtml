从SAP的发展历史看ERP-西欧思路、北美思路和中国电子商务软件应用启蒙新业态

作者:本站编辑 发布时间:2017-10-13 来源:佚名 点击数: 分享网址

关于ERP，一万个人，有一万种理解，有人认为这是一个管理软件，有人认为这是一个财务软件，有人认为这是生态链，有人认为这是一种管理咨询服务。

百科对此解释是：

ERP管理系统是现代企业管理的运行模式。它是一个在全公司范围内应用的、高度集成的系统，覆盖了客户、项目、库存和采购、供应、生产等管理工作，通过优化企业资源达到资源效益最大化。

然而，这一软件的具体思维是如何产生的呢？本文在此进行全面诠释。


一、SAP创立


SAP创立于1972年，德国。


SAP的主要创始人来自IBM，德国的IBM，而且是博士学历出身、IBM德国的高管VP。


当时全世界都是以硬件销售为主，软件走的是程序员定制开发。当时的硬件以大型机为主，非常昂贵，只有大型企业才能用得起。


当时在IBM有两派声音：一派是觉得卖硬件多利索多赚钱，软件谁想做谁做，走集成模式就OK；一派认为IBM作为全球IT领导者，必须为客户提供解决方案，而且大型客户要的就是解决方案，否则客户孤零零买台大型机干嘛，所以这派的观点是IBM也要研发软件。当然大家知道了，是IBM在80年代末期大型主机不好卖了，郭士纳1993年上台才转型软件产业走软硬件一体化解决方案。但在1972年，大部分IBM人还是赞成IBM只做硬件，把各个企业需要的千奇百怪的应用软件需求放给生态合作伙伴。


而SAP的创始人觉得软件才是企业真正想要的价值，一台空的大型主机就如同一台没有音乐磁带的录音机没啥用处。而且他们在德国做了多个单子，发现客户当时需要的企业应用软件都有一定的相似性，他们认为可以提炼出一套软件产品。


于是他们就准备开干，他们早就搞定了一家客户关系，说你先等等，等我们出去开公司了你购买我们开发的软件。于是，这家公司就购买了IBM大型主机，SAP创建了，SAP的几个创始人就开始使用客户的主机进行编程。


要知道，1972年的德国，仍然是欧洲的工业强国，当时德国制造异常强大，生产制造管理非常复杂，也需要计算机自动化来简化。德国人也非常喜欢严谨的明确的流程制度规范。而且德国处于国家零碎的欧洲（当时还没有欧盟与欧元），每个国家的语言、税制、货币计量都不同。但德国的企业却需要在欧洲各个国家开展生意。


所以当时德国企业对流程制度规范固化，对跨国家的语言/税制/货币简化处理，对生产制造复杂工序管理需求非常强烈，德国企业有强大的买单能力，SAP正好应景产生。


这样的结果就是一开始就做对了：

SAP ERP做大型企业

SAP ERP售价高

SAP ERP擅长处理国际化业务

SAP ERP擅长处理生产制造


二、SAP在美国


SAP往美国发展，是因为SAP的欧洲客户要在美国开展业务，于是SAP就顺便跟了过去。


但美国的套路和德国不一样。美国走的是全球供应链战略，美国来控制游戏规则、获取最大利益。哪个国家充当研发设计与品牌塑造国，哪个国家充当原料/能源提供国，哪个国家充当生产制造国，哪个国家充当倾销市场国，美国人玩的大，不像德国老老实实制造精密机床老老实实做顶尖制造。所以这也造成了美国本土Oracle产生的ERP擅长供应链管理却不擅长生产制造。所以，以生产制造为重心的企业建议采用SAP ERP，以采销供应链为重心的企业建议采用Oracle ERP。


也是因为美国的战略玩法，所以美国玩ERP不是买软件上软件用软件，而是以战略咨询流程咨询为切入先做企业流程重组或企业战略变革，最后才落地固化到ERP IT系统中。


而且美国有这么一条完整成熟的链条：家族产业托管基金-所有权和经营权分离-职业经理人-MBA商学院-经典管理模型教授研究-咨询顾问-ERP流程固化与数据透明。这条链，德国没有。这条链，中国更没有。


于是，SAP在美国的扩展的模式：主要以咨询公司引领，SAP ERP负责落地固化。


三、SAP在中国


SAP进入中国，也是因为跟着德国公司美国公司这些客户顺便进入中国的。因为中国的改革开放，大力发展合资公司，或者到中国来做委托加工。


对于合资公司，往往是技术转移，但老外也鸡贼，不是把技术图纸单纯的转移，而是从产品图纸、产品生产设备、产品核心零部件、设备维护工程师、设备管理IT系统、生产管理IT系统，全套引入。对于委托加工，为了做到全球产业链控制，为了做到产供销联动，所以外国公司给中国发包，也要求中国代工企业上线ERP，和外国母公司连接在一起，这样老外就能在外国做到信息得知、流程管控。


后来，不少中国企业也长大了，但是由于历史原因，要么接受了外国的投资，要么不能在国内上市而需要在欧美股市上市，为了符合外国投资者的监管、国际审计惯例，所以也都上线了SAP ERP或Oracle ERP，主要使用财务模块，主要就是为了做外国上市财报和审计，就这个核心作用。


所以，要么是依赖外国订单、要么是依赖外国技术、要么是依赖外国资本、要么是想把生意做到外国，所以才需要上SAP ERP和Oracle ERP。


四、中国能不能产生类似SAP的软件公司


依目前时间节点，我看不容易。因为大家都知道，产品都是逐步改进成熟的，不可能一开始一起步就横空出世很牛逼。所以势必刚开始时是很烂的。其实SAP在刚刚创业的时候产品也是很烂的，但是1972年ERP产品市场一片空白，而且德国企业需求非常强盛而且买单能力也超强，所以SAP ERP一开始能卖出去并且还能卖高价。但是现在中国，你让一个大型企业购买中国的一家创业公司的ERP来替换它现在使用的SAP或Oracle ERP，可能吗？不可能。你让一家创业公司的ERP卖的和SAP一样贵可能吗？也不可能。哪个老板和哪个CIO都不敢担这个历史责任。（当然，很多CIO选型SAP ERP就是私心里为了逃避未来可能的失败责任，反正我已经选型了世界上所谓的最好的ERP，咱们实施失败不是我选型失败，肯定是咱们自己不好，而不是SAP软件不对）。


那我们是否就悲观了？


不，我们是没法重复走一遍SAP当年的老路，但是我们有我们的历史机遇。那就是电子商务、产业链集群。


有什么样的消费者就有对等水平的产品制造提供商。中国制造企业很难自己主动升级改善，必须让消费者（给钱的人）倒逼，大家才愿意改进（能躺着赚钱谁想站着啊）。


恰好，中国现在处于消费升级的历史时期。我们终于拥有了所谓的一亿中产，可以形成能撼动产品制造商的影响力量了。


而且我国的电子商务在互联网风投的驱动下异军突起。有钱、有互联网聚合的巨大消费者用户人群流量、有强大的IT技术和自动化仓储物流技术、有强大的智能手机普及率、普及的4G和WIFI网络、普及的移动支付和互联网金融信贷。而且我国还有强大的丰饶甚至淤出来的日用消费品的生产制造（中国是全球日用消费品第一大国啊）。而且中国人口大量聚集在35mm等雨线以南、大量聚集在几大城市圈（华北环渤海、华东长三角、华南珠三角、西南成渝），而且区域生产制造集群非常明显。中国的仓储物流配送人口红利很强大、中国的卡车生产和电动自行车生产也很强大。中国的线下零售很薄弱。这样就造就了全球独一无二的中国电子商务盛况。


零售端的强大，就会倒逼生产制造。过去是生产制造出什么就卖什么。现在中国电商这么强大，势必要倒逼成消费者需要什么，中国电商就要求中国产品制造商生产什么。


所以在这种基础上，中国的企业ERP势必会一块块解耦。你看现在中国企业，基本已经把供应链、仓储物流配送、客服、销售渠道管理、订单管理、售后管理、MES、项目管理、人力资源管理都逐步从ERP中拆出来了，由更专业的独立软件来处理了。经典ERP，只主要使用财务模块了。


现在中国制造企业大多在积极探讨互联网+转型，尝试开展电商直销、B2B电商交易、产业链整合、产业网络服务平台、产业区域集群协同制造。这些业务的开展和IT系统的匹配，那么经典ERP会在不知不觉中都慢慢肢解了，而新一代的ERP，在电商、产业链、产业区域集群制造新业务中不断产生、打通，而形成。这属于中国的，独一无二的路径。


而且现在中国互联网也在纷纷杀入上游产业链、杀入线下，纷纷以新技术、新资本、巨大的消费者为要挟来重组中国传统产业链资源。


过去的ERP，大多是人手工录入，靠人在驱动流程往前走。现在是靠物联网传感器自动采集数据、靠人工智能视觉识别自动采集数据、靠智能产品智能设备自动采集数据、靠人工智能机器学习算法来做到智慧采购智慧选品智慧定价智慧仓储物流调度智慧供需资源匹配。也就是说，中国产生的ERP，是无形的，数据采集无形，数据展示在手机上，人只要做中间关键的一些确认操作。


过去的ERP专注的是一家企业的经营管理，最多也就是以一家龙头企业为中心打通产业上下游。而中国产生的下一代ERP，主要是以产业链服务平台为关键特征，这个产业链互联网络中有供应商、生产商、电商零售商、售后服务商、消费者，通过大数据来驱动供需自动最佳匹配。会有专门的产业链服务平台商出现，来提供订单运营、产业链资源整合、IT系统、数据沉淀和数据洞察分析，他们不是软件公司做ERP软件卖ERP软件，但是他们拥有比专业软件公司更庞大更强大的研发团队，他们的核心能力就是订单运营+IT平台+数据洞察。
